Autosaveify

GitHub


Automatic AJAX form submission with feedback and styling. See demo and examples.


Installation


Download latest autosaveify.css from:

https://github.com/Jaza/autosaveify/tree/master/css


Download latest autosaveify.js from:

https://github.com/Jaza/autosaveify/tree/master/js


Include necessary scripts and styles:

<head>
  <!-- ... -->

  <link href="path/to/bootstrap.min.css" rel="stylesheet" >

  <link href="path/to/autosaveify.css" rel="stylesheet">
</head>

<body>
  <!-- ... -->

  <script src="path/to/jquery.min.js"></script>
  <script src="path/to/underscore-min.js"></script>

  <script src="path/to/autosaveify.js"></script>
</body>

Usage


Set data-toggle="autosaveify" on a form element, along with data-save-url="http://host.name/endpoint" (if not provided, the form's action value will be used as the save URL). AJAX requests will be made using the HTTP method specified in the form's method value.

<form id="foo-form" method="POST" action="http://host.name/foo/save" data-toggle="autosaveify" data-save-url="http://host.name/foo/ajax-save">
  <!-- form elements here -->
</form>

Alternatively, call Autosaveify with a single line of JavaScript.

$('#foo-form').autosaveify(options);

Options


Options can be passed via data attributes or JavaScript.

Name data attribute type default description
fakeSuccess data-fake-success boolean false Instead of actually making AJAX requests, Autosaveify "fakes it" by simply showing the "success" icon or status box. Useful for testing the success styling when no back-end server is available.
hideOnBlur data-hide-on-blur boolean false Hides the autosaveify icon whenever an element in an Autosaveify form loses focus. Only applies when using the icon markup; when using the status box markup, the status box is always hidden on blur.
pendingDelayInMs data-pending-delay-in-ms integer 2000 How long to wait (in milliseconds) for a user to stop typing, before attempting to do an autosave.
saveUrl data-save-url path none Makes an AJAX request to this URL on save. If not provided, the form's action value is used as the save URL.
successMsgDurationInMs data-success-msg-duration-in-ms integer 5000 How long the success message should stay visible each time it shows.

Methods


.autosaveify(options)

Activates your form with Autosaveify. Accepts an optional options object.

$('#foo-form').autosaveify({
  saveUrl: 'http://host.name/foo/save'
});

.autosaveify('validate')

Checks that all required form fields have been filled in, and that no autosave is already in progress. If validation passes, shows the warning message "Unsaved changes". If validation fails, shows a warning message indicating the problem.

$('#foo-form').autosaveify('validate');

.autosaveify('save')

Saves the form via an AJAX request, after pendingDelayInMs time has elapsed. If this function is called again, for the same form, before the time has elapsed, then the timer is reset. So, as long as the user continues changing input values without pausing for more than "x" seconds, the save is not performed. Only after the user stops editing the form for more than "x" seconds does the save happen.

$('#foo-form').autosaveify('save');

.autosaveify('saveIfValid')

Calls .autosaveify('save') only if .autosaveify('validate') returns true.

$('#foo-form').autosaveify('saveIfValid');

.autosaveify('showStatus')

Shows the Autosaveify icon(s) or status box(es) for the selected form.

$('#foo-form').autosaveify('showStatus');

.autosaveify('hideStatus')

Hides the Autosaveify icon(s) or status box(es) for the selected form.

$('#foo-form').autosaveify('hideStatus');

.autosaveify('destroy')

Removes Autosaveify functionality from the selected form and removes all attached event handlers.

$('#foo-form').autosaveify('destroy');

Events


Event Type Description
save.autosaveify This event fires when the save instance method is called, but not until after pendingDelayInMs time has elapsed with no user form input.
saved.autosaveify This event is fired when the save AJAX request has completed. It fires whether the AJAX request succeeds or fails. If the fakeSuccess option is enabled, then this event is fired immediately after save.autosaveify fires.
validate.autosaveify This event fires when the validate instance method is called.
validated.autosaveify This event is fired when Autosaveify validation has completed. It fires whether the validation succeeds or fails.
